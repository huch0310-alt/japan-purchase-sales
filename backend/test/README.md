## 🚀 运行测试

### 快速开始

```bash
# 1. 配置测试环境
cp .env.test.example .env.test

# 2. 创建测试数据库
createdb japan_purchase_sales_test

# 3. 运行所有测试
npm test && npm run test:e2e

# 4. 查看覆盖率报告
open coverage/lcov-report/index.html
```

### 测试命令

```bash
# 单元测试
npm test                        # 运行所有单元测试
npm run test:watch              # 监听模式
npm run test:cov                # 生成覆盖率

# E2E测试
npm run test:e2e                # 运行所有E2E测试
npm run test:e2e:cov            # E2E覆盖率

# 特定测试
npm test -- products.service.spec.ts
npm run test:e2e -- auth.e2e-spec.ts
```

## ⚠️ 注意事项

### 1. 测试环境要求
- PostgreSQL数据库
- Node.js环境
- 测试数据库独立运行

### 2. 测试数据管理
- 测试数据会自动清理
- 不要在生产环境运行测试
- 测试数据库应与生产数据库隔离

### 3. 测试执行顺序
- 单元测试先于E2E测试
- E2E测试按模块顺序执行
- 测试之间相互独立

### 4. 已知限制
- 部分测试需要真实数据库
- 文件上传测试需要实际文件
- WebSocket测试需要额外配置
