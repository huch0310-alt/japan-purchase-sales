# 日本采销管理系统 - CI/CD流水线设计

## 概述

为日本采销管理系统构建完整的CI/CD流水线，实现代码提交到生产部署的自动化。

## 技术选型

| 组件 | 技术方案 | 理由 |
|------|----------|------|
| CI/CD平台 | GitHub Actions | 免费、开源、与GitHub集成 |
| 代码质量 | ESLint + Prettier | 代码风格统一 |
| 测试 | Jest + Supertest | 单元测试和集成测试 |
| 容器构建 | Docker BuildKit | 快速构建 |
| 部署 | Docker Compose | 容器化部署 |

## 流水线设计

### 1. 开发环境流水线

```
代码提交 → Lint检查 → 单元测试 → 构建镜像 → 推送到开发环境
```

### 2. 测试环境流水线

```
合并到main → 完整测试 → 构建镜像 → 推送到测试环境 → 自动化测试
```

### 3. 生产环境流水线

```
发布Release → 构建生产镜像 → 推送到生产环境 → 健康检查
```

## 工作流设计

### 1. CI工作流 (ci.yml)

- 触发: 代码推送和PR
- 步骤:
  1. 安装依赖
  2. 代码检查 (ESLint)
  3. 单元测试
  4. 构建检查

### 2. CD工作流 (cd.yml)

- 触发: 推送到main分支
- 步骤:
  1. 构建后端镜像
  2. 构建前端镜像
  3. 推送镜像到仓库
  4. 部署到服务器

### 3. Release工作流 (release.yml)

- 触发: 创建GitHub Release
- 步骤:
  1. 构建生产镜像
  2. 创建版本标签
  3. 部署到生产环境

## 验收标准

1. 所有代码提交自动运行CI流程
2. 代码检查和测试失败阻止合并
3. 自动构建并推送Docker镜像
4. 支持回滚到上一个版本
5. 部署过程可视化
